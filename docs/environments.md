# Environments

## Local (only environment in repo)

This project has a single environment: **local development via Laravel Sail**. There is no staging or production configuration in the repository. The app is intended to be run locally.

### How it is deployed

Laravel Sail is used as the local Docker wrapper. The `docker-compose.yml` defines three services:

| Service | Image | Role |
|---------|-------|------|
| `laravel.test` | `sail-8.3/app` (built from `vendor/laravel/sail/runtimes/8.3`) | PHP 8.3 + Nginx/Apache app server |
| `mysql` | `mysql/mysql-server:8.0` | Primary database |
| `memcached` | `memcached:alpine` | Cache backend |

**Quick start:**

```bash
composer install
cp .env.example .env
sail up -d
sail artisan key:generate
sail artisan migrate
sail artisan db:seed
sail artisan test
# open http://localhost
# login: default@example.com / password
```

### Required environment variables

| Variable | Purpose |
|----------|---------|
| `APP_NAME` | Application name (displayed in UI) |
| `APP_KEY` | Laravel encryption key (generated by `artisan key:generate`) |
| `APP_ENV` | Environment name (`local`) |
| `APP_URL` | Base URL used by Laravel for link generation |
| `DB_CONNECTION` | Database driver (`mysql`) |
| `DB_HOST` | MySQL host (set to `mysql` for Sail) |
| `DB_PORT` | MySQL port (`3306`) |
| `DB_DATABASE` | Database name |
| `DB_USERNAME` | Database user |
| `DB_PASSWORD` | Database password |
| `CACHE_DRIVER` | Cache backend (`memcached`) |
| `MEMCACHED_HOST` | Memcached host (set to `memcached` for Sail) |
| `MEMCACHED_PORT` | Memcached port (`11211`) |
| `COINMARKETCAP_API_KEY` | API key for CoinMarketCap (crypto rate fetching) |
| `TELEGRAM_BOT_TOKEN` | Telegram Bot API token (webhook URL + bot API calls) |
| `TELEGRAM_BOT_DOMAIN` | Publicly reachable domain for Telegram webhook (e.g. localxpose tunnel URL) |
| `TELEGRAM_USER_IDS` | JSON map of `{"<db_user_id>": <telegram_user_id>}` for notifications |
| `WWWUSER` / `WWWGROUP` | UID/GID passed to Sail container (typically `1000`) |

Optional debug variables:

| Variable | Purpose |
|----------|---------|
| `SAIL_XDEBUG_MODE` | Enable Xdebug (`debug` to activate) |
| `DEBUGBAR_OPEN_STORAGE` | Enable Laravel Debugbar storage viewer |

### External services

| Service | URL | Used by |
|---------|-----|---------|
| cbr-xml-daily.ru | `https://www.cbr-xml-daily.ru/daily_json.js` | `rates:get` command — USD/RUB rate |
| dolarapi.com | `https://dolarapi.com/v1/dolares/blue` | `rates:get-usd-ars` command — USD/ARS blue-market rate |
| CoinMarketCap API | `https://pro-api.coinmarketcap.com/v1/cryptocurrency/quotes/latest` | `app:get-crypto-rates` command |
| Telegram Bot API | `https://api.telegram.org` | Webhook registration, planned expense notifications |

### Scheduled commands

Managed by `app/Console/Kernel.php`. In Sail, the scheduler must be running (`sail artisan schedule:work` or a cron that calls `artisan schedule:run`).

| Command | Schedule | Purpose |
|---------|----------|---------|
| `rates:get` | Weekdays 06:00–12:00, hourly | Fetch USD/RUB rate from CBR |
| `rates:get-usd-ars` | Weekdays 12:00–18:00, hourly | Fetch USD/ARS blue rate |
| `app:get-crypto-rates` | Hourly | Fetch crypto rates from CoinMarketCap |
| `app:notify-planned-expense` | Daily at 10:00 | Send Telegram reminders for due planned expenses |
| `backup:run` | Twice daily (03:00, 15:00) | Create DB + file backup via spatie/laravel-backup |

### Deploy / rollback

There is no automated deployment pipeline. For local use:

- **Update:** pull new code, run `sail artisan migrate`, restart Sail if needed.
- **Rollback:** restore from backup. Place the backup archive in `storage/app/backup/`, then:

  ```bash
  sail artisan backup:restore --reset
  sail artisan cache:clear
  ```

### Testing

```bash
sail artisan test
```

PHPUnit configuration is in `phpunit.xml`. Tests use a separate `testing` database (created by the Sail MySQL init script at `vendor/laravel/sail/database/mysql/create-testing-database.sh`).

### Telegram Bot local testing

Use [localxpose.io](https://localxpose.io/) or a similar tunnelling tool to expose Sail's port 80 to the internet, then:

1. Set `TELEGRAM_BOT_DOMAIN` to the tunnel URL in `.env`.
2. Run `sail artisan app:bot-set-webhook` to register the webhook with Telegram.
3. Telegram user IDs can be obtained from `@userinfobot`.
